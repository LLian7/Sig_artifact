Artifact Evaluation Notes
=========================

Repository scope
----------------
This directory contains the implementation, parameter-search code, benchmark
code, cached benchmark outputs, optional native accelerators, and unit tests for
the YCSig / ValStrictISP artifact.

The repository now keeps benchmark artifacts as JSON caches and offers text or
JSON command-line output from the benchmark scripts. It does not contain LaTeX
section-generation code or checked-in .tex files.

The section mapping below is intended to help artifact evaluators connect the
paper claims to the code. The section names follow the current manuscript
organization used in this repository:
- construction / scheme description
- complexity analysis
- comparison
- implementation / other benchmarks


1. Paper-to-code mapping
------------------------

1.1 Core construction: YCSig
- yc_sig.py
  Main implementation of the YCSig one-time signature scheme.
  Includes setup, key generation, signing, verification, retry handling, and the
  mapping from accepted ValStrictISP outputs to signed leaf indices.
- crypto_utils.py
  Hash/XOF helper functions used across the construction.
- keyed_hash.py
  Keyed hash wrapper used by the construction.
- tweakable_hash.py
  Tweakable hash helper used in ADS / Merkle-related parts.
- merkle_tree.py
  Merkle tree implementation used for public-key compression and authentication.
- pprf.py
  PPRF implementation used for punctured secret derivation.
- yc_sig_native.c, setup_yc_sig_native.py
  Optional native helper for selected YCSig hot paths.

1.2 Core algorithm: ValStrictISP
- val_strict_isp.py
  Main implementation of the simplified-windowed ValStrictISP algorithm.
  Includes:
  - parameter class ISPParameters
  - window bounds / feasibility guard
  - sampling / deterministic XOF-driven subset generation
  - the public val_strict_isp(...) interface
- val_strict_isp_native.c, setup_val_strict_isp_native.py
  Optional native helper for selected ValStrictISP hot paths.

1.3 Complexity analysis and parameter search
- search_ycsig_sigsize.py
  Exact search for admissible (HashLen, MaxGValue, PartitionNum, WindowRadius)
  rows under retry/UCR constraints. This script is the code counterpart for the
  paper's complexity-analysis and parameter-selection sections.
- params_windowed_comparison_current.json
- params_windowed_comparison_128_160.json
- params_windowed_comparison_128_160_verify_tiebreak.json
- params_windowed_implementation_192_current.json
- params_case1_w4_partition_verify_sig.json
- params_case1_w4_partition_verify_sig_retry40ish.json
- params_case2_w4_partition_verify_sig_retry40ish.json
  Cached parameter selections used for comparison / implementation tables and
  parameter-search checks.

1.4 Comparison section
- benchmark_ycsig.py
  Shared analytic/empirical benchmark driver for YCSig.
- benchmark_ycsig_ops.py
  Operation-count benchmark with hash-equivalent accounting.
- benchmark_ycsig_table.py
  Encodes the paper rows used by benchmark-table helpers.
- benchmark_ycsig_table_comparison.py
  Recomputes the YCSig rows used in the comparison table from implementation
  counters and supports JSON/text output.

1.5 Implementation / Other Benchmarks section
- benchmark_ycsig_cycles.py
  Cycle benchmark driver; supports retry/sign/verify timing, optional xctrace
  hardware-counter measurement, and optional tracemalloc-based peak-memory
  tracking.
- benchmark_true_cycles.py, benchmark_true_cycles_worker.py
  Shared xctrace hardware-counter utilities and worker process entry point.
- benchmark_val_strict_isp_cycles.py
  Standalone cycle benchmark for ValStrictISP. Reports the multiplicity-profile
  cost, acceptance-check cost, and end-to-end accepted/rejected ValStrictISP
  cost.
- benchmark_ycsig_table_ops.py
  Table-oriented wrapper for operation-count benchmarks.
- benchmark_ycsig_table_cycles.py
  Table-oriented wrapper for cycle and memory benchmarks.
- benchmark_ycsig_table_dual.py
  Combined wrapper that reports both operation-count and cycle-table results.
- refresh_other_benchmarks_data.py
  Refreshes the cached implementation benchmark JSON files. This script no
  longer generates LaTeX output.
- implementation_results_128_160_current.json
- implementation_results_192_current.json
- implementation_w16_block_first_results.json
- val_strict_isp_cycles_results_current.json
- val_strict_isp_cycles_case12_w4_results_current.json
- val_strict_isp_cycles_case12_multiw_results_current.json
- smoke_implementation_results_128_160.json
- smoke_implementation_results_192.json
- smoke_val_strict_isp_cycles_case12_w4.json
  Cached implementation-level benchmark outputs and small smoke-run outputs.

1.6 Contribution paragraph / high-level claims
There is no separate executable "contribution paragraph" generator in the

2. Reproduction guide
---------------------

2.1 Sanity check / main correctness tests
Run:
python3 -m unittest test_yc_sig.py test_val_strict_isp.py

2.2 Full relevant test suite used during repository maintenance
Run:
python3 -m unittest \
  test_yc_sig.py \
  test_val_strict_isp.py \
  test_benchmark_ycsig.py \
  test_benchmark_ycsig_ops.py \
  test_benchmark_ycsig_cycles.py \
  test_benchmark_ycsig_table.py \
  test_benchmark_ycsig_table_dual.py \
  test_benchmark_val_strict_isp_cycles.py \
  test_benchmark_true_cycles.py \
  test_search_ycsig_sigsize.py \
  test_merkle_tree.py \
  test_pprf.py \
  test_operation_counter.py

2.3 Parameter search
Example:
python3 search_ycsig_sigsize.py --format text

JSON export:
python3 search_ycsig_sigsize.py --format json

2.4 Comparison-table recomputation
Example:
python3 benchmark_ycsig_table_comparison.py \
  --repetitions 5000 \
  --params-json params_windowed_comparison_current.json \
  --format text

2.5 Implementation benchmark recomputation
Operation-count table:
python3 benchmark_ycsig_table_ops.py \
  --samples 32 \
  --repetitions 40 \
  --format text

Cycle/memory table:
python3 benchmark_ycsig_table_cycles.py \
  --samples 32 \
  --repetitions 40 \
  --cpu-frequency-ghz 3.49 \
  --measure-memory \
  --format text

Combined wrapper:
python3 benchmark_ycsig_table_dual.py \
  --samples 32 \
  --repetitions 40 \
  --cpu-frequency-ghz 3.49 \
  --mode both \
  --format text

2.6 Standalone ValStrictISP cycle benchmark
Example:
python3 benchmark_val_strict_isp_cycles.py \
  --hash-len 256 \
  --max-g-bit 16 \
  --partition-num 2 \
  --window-radius 0 \
  --accepted-samples 256 \
  --rejected-samples 64 \
  --repetitions 40 \
  --cpu-frequency-ghz 3.49 \
  --format text

2.7 Refresh cached implementation benchmark JSON
Example:
python3 refresh_other_benchmarks_data.py \
  --samples 32 \
  --repetitions 40 \
  --cpu-frequency-ghz 3.49 \
  --val-accepted-samples 32 \
  --val-rejected-samples 8 \
  --val-repetitions 10

This writes the current implementation benchmark JSON caches only:
- implementation_results_128_160_current.json
- implementation_results_192_current.json
- val_strict_isp_cycles_case12_multiw_results_current.json


3. Experimental environment
---------------------------

3.1 Programming language
- Python implementation
- Tested with CPython 3.14.4

3.2 Operating system
- Tested on macOS 26.4.1
- Kernel / platform string observed during maintenance:
  Darwin 25.4.0, arm64

3.3 Compiler / build environment
- The primary artifact code is Python and relies on the standard library.
- Optional native helpers can be built from yc_sig_native.c and
  val_strict_isp_native.c with the corresponding setup scripts.
- The Python interpreter in the tested environment reports:
  Clang 17.0.0 (clang-1700.6.4.2)

3.4 Python package requirements
- No third-party Python packages are required by the runtime code or the test
  suite in this directory.
- Imported dependencies are either:
  - Python standard-library modules
  - local repository modules in this directory

3.5 Standard-library features used
- argparse
- dataclasses
- functools
- gc
- hashlib
- json
- math
- random
- statistics
- struct
- subprocess
- time
- tracemalloc
- typing
- unittest

3.6 Benchmark environment notes
- Cycle benchmarks can use per-thread CPU time estimates or macOS xctrace
  hardware CPU counters.
- If automatic CPU-frequency detection is unsuitable on a target machine,
  explicitly pass:
  --cpu-frequency-ghz <value>
- The paper-style implementation tables in this repository were maintained with
  3.49 GHz as the nominal CPU frequency.
- xctrace-based measurements are macOS-specific and may require command-line
  developer tools / Instruments support on the evaluation machine.
